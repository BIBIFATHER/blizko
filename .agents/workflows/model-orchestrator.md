# Model Orchestrator

## Правило

Перед каждой крупной задачей оцени: **какая модель лучше справится?** Если текущая не оптимальна — предложи переключение.

|---|---|---|---|
| **Claude Opus 4.6 (Thinking)** | Код, архитектура, сложный дебаг | 💰💰💰 | Сложные фичи, рефакторинг, multi-file edits |

| **Gemini 3.1 Pro (Low)** | Аналитика (экономия) | 💰 | Простая аналитика, саммари, обзоры |
| **Gemini 3 Flash** | Быстрые ответы, boilerplate | ⚡ | Простые правки, rename, форматирование |
| **GPT 5.3** | Креатив, копирайтинг, tone of voice | 💰💰 | Слоганы, UX-тексты, сценарии Reels, email |

## Когда предлагать переключение

1. **Задача аналитическая** → «💡 Gemini 3.1 Pro (High) лучше справится с анализом»
2. **Нужны свежие данные** → «💡 Perplexity найдёт актуальные цифры»
3. **Креатив/тексты** → «💡 GPT 5.3 сильнее в копирайтинге»
4. **Простая механика** → «💡 Gemini Flash справится за 10% стоимости»
5. **Сложный код** → оставайся на Claude Opus 4.6

## Формат предложения

```
💡 Для [задача] лучше переключиться на [модель]:
— [причина 1]
— [причина 2]
Экономия: ~[X]x по токенам
Хотите переключиться?
```

## Передача контекста

При переключении подготовь промпт:

- Что уже сделано (кратко)
- Текущая задача
- Ключевые файлы
- Ожидаемый формат результата
