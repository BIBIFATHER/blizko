# Blizko Ops (Draft)

## Goal
Описать операционный цикл: модерация, триаж, статусы, метрики, SLA.

## Статусы (из кода)
**parents**
- `new` → `in_review` → `approved` / `rejected`
- `resubmitted` (событие в changeLog при повторной отправке)

**nannies** (предложение)
- `draft` → `in_review` → `approved` / `rejected`

## Правила триажа (черновик A/B/C)
- **A**: срочно (24–48ч), ребёнок <3 лет, нужен полный день/ночь
- **B**: стандарт (3–7 дней), частичная занятость
- **C**: низкий приоритет (>7 дней), без срочности

## SLA (предложение)
- A: первый ответ ≤2 часа
- B: ≤8 часов
- C: ≤24 часа

## Ежедневные KPI (минимум)
- New parents / day
- % responded < SLA
- % approved
- # matches / day

## Ритуалы
- 09:00 план (проверка очереди + приоритеты)
- 18:00 итоги (что закрыто, что блокирует)
