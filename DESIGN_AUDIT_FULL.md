# Blizko — Design Audit (Full)

> Сводный аудит по реальным экранам (Relay) + приоритеты P0/P1/P2.

## P0 — критично (сразу влияет на конверсию/доверие)
1) **Explainability + Safety** рядом с ключевым действием
   - На экранах результата/заказа/оплаты обязательно: «почему подходит» + «что проверено».
2) **Один главный CTA** на каждом экране
   - Все вторичные действия — вторичными кнопками.
3) **Статусы и ожидания**
   - Явный статус заявки и ETA (подбор/проверка/подтверждение).

## P1 — важно (качество, удержание)
1) **Прогресс заполнения профиля** (родитель/няня)
2) **Единый стиль CTA и бейджей**
3) **Улучшение пустых состояний** (что делать дальше)

## P2 — полировка
1) Микротексты (снижение тревожности)
2) Унификация терминов (Humanity+/совместимость)
3) Мелкие UI‑анимации

# Экран‑за‑экраном

## 1) Home (Landing)
**Проблема:** недостаточно явной механики доверия в hero.
**Решение:** добавить 3‑шаговый flow: Запрос → Анализ → Подбор + маленький safety‑блок.

## 2) Заявка на подбор (родитель)
**Проблема:** длинный экран, нет прогресса и иерархии.
**Решение:** разбить на 3–4 шага + progress; hard‑требования отдельно.

## 3) Success / «Спасибо, мы рядом!»
**Проблема:** нет следующего действия (что делать дальше?).
**Решение:** явный CTA: «Перейти в личный кабинет» / «Посмотреть статус».

## 4) Профиль родителя
**Проблема:** нет прогресса заполнения, слабая иерархия карточек заявок.
**Решение:** прогресс‑бар + CTA “дополнить профиль”; карточки статусов с бейджами.

## 5) Заказы родителя + чат
**Проблема:** нет быстрых действий и гарантий рядом с заказом.
**Решение:** быстрые CTA (подтвердить/перенести), блок safety‑proof и таймлайн.

## 6) Отзыв
**Проблема:** модалка без мотивации/подсказок.
**Решение:** мини‑подсказка «отзывы повышают качество подбора».

## 7) Админка
**Проблема:** перегруз действий.
**Решение:** разграничить primary (Одобрить/Отклонить) и вторичные (экспорт/массовые).

## 8) Профиль няни
**Проблема:** нет прогресса/ценности; оплата смешана с профилем.
**Решение:** прогресс заполнения + отдельный блок «Почему это важно».

## 9) Календарь няни
**Проблема:** статусы (R/✕) неочевидны.
**Решение:** легенда «Свободно/Резерв/Занято» с цветами.

## 10) Оплата (активация профиля)
**Проблема:** слишком много элементов в одном модальном окне.
**Решение:** 1 главный CTA + короткое объяснение «что входит». Опции оплаты ниже.

## 11) Заявки няни
**Проблема:** нет ясности «что будет после принятия».
**Решение:** добавить micro‑info «после принятия: чат + подтверждения T‑24/T‑3». 

## 12) Отзывы няни
**Проблема:** пустое состояние без мотивации.
**Решение:** CTA «получить первые отзывы» + объяснение пользы.
