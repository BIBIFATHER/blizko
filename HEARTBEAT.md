# HEARTBEAT.md

## Heartbeat Checklist

Run only if there is no active higher-priority user task.

### Morning

- Check current workspace status and surface any blocking build, test, or deploy issue.
- Review recent memory files and note anything worth promoting to long-term memory.
- Check whether today's work aligns with the top product priorities.

### Midday

- Look for stale branches, broken local workflows, or unrecorded decisions.
- Inspect whether current work should create a changelog or memory update.
- If the user is likely to waste motion by manually searching across tools, suggest one exact plugin and one exact next step.

### Evening

- Prompt a short ACT-style reflection: what moved the product forward, what created drag, what should be removed tomorrow.
- Suggest one cleanup or automation that would reduce cognitive load.
- Suggest one plugin-based shortcut only if it would clearly reduce tomorrow's friction.

## Quiet Rules

- If nothing important changed, respond with `HEARTBEAT_OK`.
- Avoid late-night nudges unless something is actually urgent.
