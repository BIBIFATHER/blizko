## Что верифицировано vs требует юриста

**Verified rule (из первоисточников 152-ФЗ; текущая редакция в этой сессии не
переверена повторно):**
- ст. 18 — локализация первичной записи ПДн граждан РФ в РФ.
- ст. 22 — уведомление оператора (до начала обработки, кроме исключений).
- ст. 12 — трансграничная передача (отдельный режим уведомления с 01.03.2023).
- ст. 10 — спецкатегории (отдельное письменное согласие).
- ст. 11 — биометрия (отдельный режим).

**Application (к фактам Blizko):**
- Обычные данные (имя/телефон/город/расписание) на Timeweb-РУ-плоскости +
  уведомление РКН + согласия + Политика = защитимый минимум для пилота.
- Геокод/AI/Telegram-egress закрыты гейтами — трансгранично сейчас не происходит.

**Counsel required (нужен человек-юрист до открытия потока):**
- Спецкатегории: паспорт/медкнижка/справки (ст.10) — текст согласия + основание + УЗ.
- Биометрия: используется ли фото/видео для идентификации (ст.11) — если да, режим меняется.
- Данные детей — корректное согласие законного представителя + минимизация.
- Точный уровень защищённости ИСПДн (УЗ) — определяет ИБ-специалист (memo §9.3).
- Трансграничная передача (Gemini/Supabase EU/Sentry/Nominatim) — ст.12-оценка + процесс РКН до открытия.

## Минимальный self-serve путь (без платного юриста)
1. Закрыть открытую регистрацию (BLI-121, владелец).
2. RU data plane на Timeweb для обычных данных (Фаза 3 PoC).
3. Самоподача уведомления РКН (ст.22) до реальной обработки — бесплатно.
4. Опубликовать полную Политику ПДн (черновик готовлю).
5. Согласия для обычных данных (черновики готовлю).
6. Спецкатегории/трансгранично — оставить закрытыми до человека-юриста.
