# Curator Operating Script — первая семья → первый матч

Операционный runbook для ручного кураторского подбора в v1-пилоте: от новой заявки
родителя до выдачи 2–3 проверенных нянь с понятными причинами и фиксацией исхода.

> Принцип: решение — за куратором и семьёй; система/AI лишь помогает (контекст, сигналы,
> не теряем детали). Выдаём «подходящих», не «доступных». Безопасность > скорость.

## SLA (пилот)

- Первый ответ семье: ≤ 4 ч (рабочее время).
- Shortlist готов: ≤ 24 ч с момента заявки.

## Шаги

1. **Приём.** Новая заявка → вкладка «Куратор» / AdminParentsTab (статус `new`).
   Уведомление — в Telegram-канал админа.
2. **Контекст семьи.** Прочитать: возраст ребёнка, график, бюджет, район, рассказ семьи
   (`analysisNotes`/comment), требования, риск-профиль. Мало данных → связаться, уточнить.
3. **Подбор из пула.** Фильтр нянь: верифицированные, `ready_for_match`, по району /
   графику / опыту. При необходимости открыть документы (signed URL в админке).
4. **Shortlist 2–3.** Для каждого кандидата заполнить:
   - «Почему именно эта няня» — 1–2 предложения по контексту семьи;
   - «Что обсудить до первого выхода» — риск-сигналы, ожидания.
5. **Выдача.** Отправить shortlist семье. Зафиксировать факт выдачи (см. метрики —
   событие `shortlist_delivered`).
6. **Хэндофф.** Связать семью ↔ няню, обозначить следующий шаг (знакомство).
7. **Follow-up.** После первой встречи собрать обратную связь (подошла / нет / почему)
   и **зафиксировать исход** — это наполняет `matching_outcomes` (learning loop).

## Edge cases

- **Нет подходящих** → состояние «Расширяем поиск» (empty match-results). Слабых не
  выдавать; вернуться, когда появится кандидат.
- **Семья молчит** → 1 напоминание через 48 ч, затем архив.
- **Няня отказалась** → заменить кандидата, обновить исход.
