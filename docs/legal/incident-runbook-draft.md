# Реагирование на инциденты ПДн — ЧЕРНОВИК runbook (BLI-123)

Дата: 2026-06-18. **Черновик для владельца**, не юридическое заключение. Внутренний
порядок на случай инцидента с ПДн (152-ФЗ ст. 21). Действует с момента допуска
реальных пользователей. Точные сроки/форму уведомления РКН сверять с актуальной
страницей [РКН incidents](https://pd.rkn.gov.ru/incidents/) — **могут меняться**.

## Что считается инцидентом
Неправомерный/случайный доступ, передача, утечка, изменение, блокирование,
удаление, уничтожение ПДн; компрометация ключей/доступов; утечка через процессора.

## Порядок действий
1. **Зафиксировать** (время, что произошло, какие данные/субъекты, источник).
2. **Локализовать** — отозвать сессии/ключи, закрыть доступ, остановить утечку.
3. **Оценить** объём: категории данных, число субъектов, спецкатегории/дети.
4. **Уведомить РКН** в установленный срок (сверить актуальный срок и форму на
   портале РКН incidents — **уточнить, не из памяти**). `[НУЖЕН ЮРИСТ для срока/формы]`.
5. **Уведомить субъектов**, если требуется.
6. **Устранить** причину, проверить процессоров, обновить меры.
7. **Записать** в журнал инцидентов + обновить `RISK_REGISTER.md` / реестры.

## Подготовка (до пилота)
- [ ] Контакт для уведомлений (оператор) определён.
- [ ] Журнал инцидентов заведён.
- [ ] Процедура отзыва сессий/ротации ключей проверена.
- [ ] Сроки/форма уведомления РКН сверены с актуальным порталом (юрист/ИБ).

`[НУЖЕН ЮРИСТ/ИБ]`: точные сроки уведомления РКН и субъектов, форма, пороги.
