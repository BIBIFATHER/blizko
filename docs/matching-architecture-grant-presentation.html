Grant narrative · impact, safety, scalability
Blizko · grant presentation

AI matching infrastructure for safer, more stable childcare

Цель проекта — снизить количество ошибочных подборов и повысить устойчивость найма между семьями и нянями за счет прозрачной, самообучающейся системы матчинга.

Social value

Меньше неудачных наймов, меньше стресса для семьи и ребенка, выше устойчивость сотрудничества.

Technical value

От explainable heuristics к responsible AI pipeline с feedback loops и auditability.

Economic value

Ниже cost of mismatch, выше retention, меньше повторных поисков и ручной поддержки.

Problem framing

Подбор няни — это не search problem. Это bilateral trust decision с delayed feedback, sparse labels и high emotional cost of mismatch.

Primary outcome30-day retention
Secondary outcomeshire rate, churn, time-to-match
AI principleLLM explains, pipeline decides
Need

Почему обычные фильтры здесь не решают задачу

Current market failure

  • Формальные совпадения по району и бюджету не гарантируют реального поведенческого fit.
  • Причины неудачи часто “мягкие”: стиль общения, нагрузка коммуникации, дисциплина, тревожность.
  • Негативный исход дорого обходится: время, тревога, нарушение рутины ребенка.

Grant rationale

  • Нужна инфраструктура, которая превращает разрозненные сигналы в measurable quality improvements.
  • Нужна ответственная AI-архитектура с объяснимостью, fairness и rollback.
  • Нужна data flywheel, который со временем повышает качество сервиса для всех участников рынка.
Architecture

Production-ready matching pipeline

1

Filter

Hard constraints: budget, quality floor, safety, blacklist cooldown.

2

Retrieve

Candidate set 20–50. Сначала explainable heuristics, позже embedding retrieval.

3

Rank

Hybrid score from heuristic prior, learned signals, retention proxy.

4

Explore

Contextual bandit with protected top-2 and controlled novelty.

5

Explain

LLM writes warm, human-facing rationale only after deterministic ranking.

Scoring model

Hybrid scorer with safe transition from MVP to learning system

Формула designed to preserve reliability at low data volumes and gradually unlock personalization.

Final score = α × Heuristic prior + β × Learned score + γ × Retention proxy

Heuristic layer never disappears fully; it remains the safety rail and rollback path.

Stage 0–1

Build labels, validate logging, improve explanations.

Stage 2

Personalized weighting after stable feedback and retention labels.

Stage 3

Retrieval + learned ranker at scale, still with fallback governance.

Why fund this

Grant capital de-risks the data and reliability layer before full commercial compounding.

Feedback loop

Data flywheel: from user actions to measurable system improvement

Behavioral events

Impressions, profile opens, dwell time, chat starts, reply latency, booking creation.

Post-hire signals

Day-3, Day-5, Day-7 micro-surveys, cancellation reason, repeat booking or replacement.

Learning outputs

Updated weights, family preference memory, churn taxonomy, insight summaries.

Public benefit

Each additional labeled outcome increases service quality and reduces future mismatch risk.

Critical instrumentation

request_id, shown_position, propensity score, model version, factor snapshot, explore flag, outcome labels. Без этого невозможна honest evaluation of impact.

Responsible AI

Fairness, transparency, and governance

No sensitive features

Возраст, этничность, фото и иные прямые чувствительные признаки не входят в scoring.

Proxy audits

Weekly review on geo, language, education, price bands to detect proxy bias drift.

Human review

Первичные insights и policy shifts проходят ручной review before promotion.

Rollback

Previous weights and policy versions are preserved to prevent unsafe model drift.

Evaluation method

  • Shadow mode validation before user-facing policy change.
  • A/B tests for each promotion stage.
  • Calibration and fairness checks before rollout expansion.
  • Retention used as final impact metric, not clicks alone.
Rollout plan

Phased implementation roadmap

Stage 0

Shadow logging, factor capture, exploration telemetry, baseline monitoring.

Stage 1

Outcome tracking, insight generation, event wiring, first micro-survey loop.

Stage 2

Personalized ranking, conservative weight updates, retention proxy model.

Stage 3

Embedding retrieval and stronger learned ranker with full observability.

Use of grant funding

Funding accelerates the non-glamorous but high-leverage layer: outcome instrumentation, responsible AI governance, survey loop, evaluation stack, and reliable deployment of matching intelligence.
