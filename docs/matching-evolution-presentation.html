От жесткой логики и эвристик к самообучающейся системе удержания. Два этапа, обеспечивающие мгновенный старт и бесконечное совершенствование.

Два агрегатных состояния

  • State 1: Текущее (TS + Heuristics + LLM)
    Прозрачные веса, хард-фильтры бюджета и детекция рисков. Идеально для старта и предсказуемости.
  • Мост: Shadow Scoring
    Сбор сырых баллов и реальных решений пользователей для тренировки будущей модели.
  • State 2: Будущее (ML + Contextual Bandits)
    Обучение на отказах и метриках удержания. Нейросеть прогнозирует успешность сотрудничества.

    Gemini анализирует профиль няни и потребности семьи на лету, объясняя выбор теплым, человечным языком ("Объяснение Анны").

    Мост перехода

    Как мы готовим датасет для ML

    Текущая система не просто распределяет нянь — она выступает движком сбора данных для будущей Machine Learning архитектуры.

    Shadow Scoring

    Каждый скоринг тихо записывается в базу (набор весов + предсказание алгоритма). Мы видим, какие оценки давала эвристика.

    ε-Greedy Injection

    В 10% случаев система подкидывает "Wildcard" кандидата. Это позволяет собирать контр-интуитивные данные (exploration).

    →
    Retention Data

    Метч логгируется вместе с исходом: состоялся ли найм и осталась ли няня через месяц. Это — разметка для будущей модели.

    Этап 2: Будущее (Целевая архитектура)

    Самообучающийся Матчинг

    Переход от Rule-based шортлиста к системе, которая учится на каждом показе, найме и отказе.

    • Контекстные бандиты
      Система исследует новые гипотезы (explore) в безопасной зоне, максимизируя отдачу (exploit) для top-2 кандидатов.
    • Модель удержания (LTV / 30d Retention)
      Цель системы меняется с "получить клик / открытие чата" на "обеспечить долгосрочный контракт".
    • Векторные вложения (Embeddings)
      Семантический поиск по неочевидным признакам из отзывов и скрытой таксономии, невидимой простой эвристике.

    Почему это работает?

    ML-эволюция возможна только благодаря честной сборке данных на Этапе 1. Мы не запускаем "глупый" ИИ на старте — мы даем отличные результаты правилами, обучая ML на эталонном поведении с первого дня платформы.
