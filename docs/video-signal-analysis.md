# Video Signal Analysis

> Цель: использовать видео-визитку няни как дополнительный trust layer, не превращая продукт в псевдонауку.

## 1. Что можно делать

AI может помогать структурировать **наблюдаемые сигналы**:

- ясность речи
- темп речи
- связность ответа
- уверенность подачи
- полнота ответа на вопрос
- визуальная аккуратность самопрезентации

Это допустимо использовать как assistive layer для модерации и trust UX.

## 2. Чего нельзя обещать

Нельзя обещать пользователю, что система по видео reliably определяет:

- эмпатию
- ответственность
- искренность
- скрытую агрессию
- психологическую пригодность

Это слишком сильные и плохо верифицируемые выводы.

## 3. Правильная продуктовая формулировка

Не:

`AI определяет эмпатию и ответственность по интонации и реакциям`

А:

`AI помогает структурировать наблюдаемые коммуникационные сигналы из видео-визитки и выделять анкеты, которые стоит проверить внимательнее`

## 4. Безопасная архитектура

### Input

- видео-визитка 20-40 секунд
- один и тот же вопрос для всех кандидатов
- опциональная расшифровка речи

### Extracted features

- transcript completeness
- answer relevance
- speaking pace
- excessive pause rate
- audio clarity
- face present / camera stable
- sentiment-lite only at presentation level, not personality diagnosis

### Output

- `video_quality_score`
- `communication_clarity_score`
- `answer_completeness_score`
- `moderation_flags[]`
- краткое explainable summary для оператора

### Human-in-the-loop

Любое решение высокого доверия:

- approve
- reject
- quality_approved

должно приниматься человеком, не моделью.

## 5. Где это реально полезно

- приоритизация ручной модерации
- улучшение trust badge
- объяснение родителю, почему профиль выглядит более зрелым и понятным
- повышение качества first-contact

## 6. Где это не должно быть source of truth

- блокировка няни
- диагнозы или оценки личности
- автоматический вывод "надежная / ненадежная"
- финальное решение о допуске без других сигналов

## 7. MVP plan

### Phase 1

- сохранить видео
- делать transcript
- считать simple communication metrics
- показывать summary только администратору

### Phase 2

- добавить explainable badge для родителей
- учитывать video signal как слабый фактор в quality score

### Phase 3

- включить correlation analysis:
  - какие video signals связаны с reply speed
  - какие video signals связаны с repeat bookings
  - какие video signals связаны с complaints

Только после накопления исходов можно усиливать влияние этого слоя.
