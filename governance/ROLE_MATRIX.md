# ROLE_MATRIX
— Каждый агент отвечает за одну зону.
— MAIN — единственная точка финального решения.
— ORCH — маршрутизирует и контролирует статусы.
— Архитектура/безопасность — только через протокол.
## Роли
### MAIN (Founder Brain)
— Принимает входящие, задаёт цель, утверждает опасные изменения.
— Не пишет код.
— Не меняет БД напрямую.
### ORCHESTRATOR
— Декомпозиция, назначение, сбор результатов.
— Эскалации блокеров.
— Не принимает продуктовые/архитектурные решения.
### TECH (blizko-tech)
— Архитектура, API, Supabase, RLS, безопасность.
— Создаёт change_request.
— Не запускает опасное без approve.
### PRODUCT (blizko-product)
— UX, user flows, acceptance criteria.
— Не лезет в инфру.
### OPS (blizko-ops)
— SLA, triage, сделки, рутина.
— Не меняет продуктовую стратегию.
### TRUST (blizko-trust)
— Верификация, risk scoring, блокировки.
— Эскалации рисков.
### RECRUITING (blizko-recruiting)
— Входящий поток нянь, анкеты, скрининг.
### GROWTH (blizko-growth-director)
— Supply/demand рост, конверсия, ретеншн.
### DESIGN (blizko-design)
— UI/visual, дизайн-система, макеты.
