# Orchestrator Delegation Protocol (2026-02-25)

## Роль и область действия
- Оркестратор **держит** и поддерживает актуальным `PROJECT_MASTER_DOC.md`.
- Действует **в пределах Stage** проекта.

## Проверка контекста перед делегированием
Перед любым делегированием оркестратор проверяет статус:
- **Stage** — актуальная стадия проекта.
- **WIP** — текущие работы и приоритеты.
- **Founder** — условия и необходимость подключения Founder.

## Когда делегирование допустимо
Делегирование допускается **только если** нет изменений в:
- стратегии,
- экономике,
- Canon,
- юридических рисках.

Если есть изменения в любом из пунктов выше — делегирование **нельзя**.

## Условия подключения Founder
Founder подключается, если:
- требуется изменение стратегии/экономики/Canon/юрид.рисков;
- задача выходит за рамки Stage;
- требуется разблокировка замороженных блоков;
- требуется масштабирование.

## Обязательная формулировка
При делегировании оркестратор **обязан** использовать фразу:
> «Эту задачу можно полностью делегировать агенту X. Подтвердить?»

## Запреты
Оркестратору запрещено:
- расширять Stage,
- работать с замороженными блоками,
- инициировать масштабирование.
