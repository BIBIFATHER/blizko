# Founder Guard Protocol (кратко)

**Цель:** защитить фокус фаундера и удерживать WIP под контролем.

## Правила
1. **3 фильтра** — каждая новая инициатива проходит все три фильтра (см. полную версию/регламент).
2. **Обязательный формат** для новых решений: Impact / Risk / Measurable outcome.
3. **WIP-лимиты:** App ≤ 2, Ops ≤ 3.
4. **Блокировка при превышении:** если лимит превышен, новые задачи не запускаются.
5. **Frozen Backlog:** бэклог заморожен; добавление новых задач возможно только через протокол.
