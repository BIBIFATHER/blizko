# Stage 0 — Execution (from Joe recommendations)

## APP P0-1 — ParentForm 2-step flow + submit after Step1
- **Impact:** Снижение трения в форме, быстрее получаем лид; больше ранних submit.
- **Risk:** Пользователь не поймёт, что submit уже произошёл; возможная путаница с ожиданиями/обратной связью.
- **Measurable outcome:** +15–25% completion rate формы; время до первого submit ↓ на 30–40%.

## APP P0-2 — Budget simplification
- **Impact:** Упрощение выбора бюджета → меньше отказов и ошибок, выше конверсия в submit.
- **Risk:** Слишком грубая шкала ухудшит качество матчинга или ожидания клиента.
- **Measurable outcome:** −20% drop-off на шаге бюджета; +10–15% общий submit rate.

## OPS P0-1 — Funnel handling for new submits
- **Impact:** Быстрый контакт с лидом → выше шанс закрытия на оплату.
- **Risk:** Нечёткий процесс увеличит хаос/пропуски и нагрузку.
- **Measurable outcome:** 90% лидов получают первый контакт ≤ 2 часов; +10% конверсия submit → call.
