# BIG_CLAW_SYSTEM — Canon Hierarchy

## Canon Hierarchy (Source of Truth)

**Никаких пересечений ответственности.** Каждый документ отвечает только за свою область.

1) **core/BLIZKO_MASTER_PLAN.md** — *куда мы идём*
   - приоритеты
   - фаза
   - стратегический фокус
   - главный файл по направлению и целям

2) **core/BIG_CLAW_SYSTEM.md** — *как мы работаем*
   - операционная модель: **Main → Orchestrator → Sub‑agents**
   - write‑scope (кто что может менять)
   - протокол делегирования
   - правила логирования решений

3) **core/ARCHITECTURE_CANON.md** — *как мы строим*
   - архитектурные решения
   - ограничения стека
   - правила безопасности

4) **core/METRICS_GLOSSARY.md** — *как мы измеряем*
   - определения метрик
   - формулы
   - связь с воронкой **contacted → responded → approved → matched → deal_done**

### Принцип управления изменениями
- **MASTER_PLAN** может менять приоритеты и фазу.
- **SYSTEM** определяет процесс и роли.
- **ARCHITECTURE** задаёт технические ограничения и безопасность.
- **METRICS** фиксирует определения и формулы метрик.

Любые изменения — только в пределах своей зоны ответственности.
