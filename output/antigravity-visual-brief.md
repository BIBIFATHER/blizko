# Blizko Antigravity Visual Brief

## Purpose

Use this brief in Antigravity to generate a redesign direction for Blizko without changing product logic.

Goal:
- redesign the visual language
- improve trust, calmness, clarity, and conversion
- keep the product essence intact

Spec lock:
- Blizko is not a generic marketplace
- Blizko is not a cute parenting app
- Blizko is not a clinical healthcare app
- Blizko is a trust-first nanny selection product

North Star:

`Blizko should feel like a trust-first premium family marketplace with healthcare-grade reassurance, calm decision architecture, and human warmth without childishness.`

## Product Context

Blizko is a nanny matching service.

It sells:
- trust
- safety
- predictability
- explainability
- peace of mind for families

Core roles:
- family
- nanny

Core flows:
- onboarding
- role selection
- candidate feed
- nanny card
- detailed nanny profile
- shortlist / compare / match
- booking / visit / deal flow
- chat / support
- parent dashboard
- nanny dashboard

## Visual Direction

Recommended direction:

`Trust-first Family Marketplace`

Blend in:
- warmth and softness from premium family-care products
- trust architecture from telemedicine and caregiver apps

Do not blend in:
- playful mommy-blog aesthetics
- startup gradient energy
- fintech harshness
- glassmorphism for effect

## What To Emphasize

- calm premium trust
- warm neutral surfaces
- strong profile trust architecture
- clear next steps
- layered trust markers
- explainability
- shortlist and compare as first-class UX

## What To Avoid

- childish illustration-heavy UI
- candy pastels
- dark mode cyber vibes
- neon accents
- luxury-lifestyle excess
- decorative trust badges with no evidence
- overloaded generic marketplace cards

## Design System Direction

Palette:
- base: ivory, warm stone, paper beige
- accents: muted teal, calm blue, olive green
- warning: soft amber
- error: restrained brick red

Typography:
- restrained editorial serif for high-trust headings
- clean sans for body and controls

Surface logic:
- soft depth
- warm white cards
- quiet borders
- generous spacing

Component logic:
- cards are trust summaries, not decorative tiles
- one primary CTA per screen
- one safe secondary CTA
- support action always visible in key trust moments

Trust atoms:
- verified identity
- reviewed documents
- response speed
- last active
- review count
- experience checked
- why-this-fit explanation

## Screens To Generate

Generate 3 polished mobile screens:

1. Detailed nanny profile
2. Shortlist / compare / match
3. Parent dashboard

Optional fourth:
- onboarding / welcome

## Screen Intent

Detailed nanny profile:
- should feel like a confidence architecture
- trust summary above the fold
- clear next actions

Shortlist / compare:
- should reduce ambiguity
- should help parents make a calm decision
- should explicitly explain differences

Parent dashboard:
- should show status, next steps, support, and shortlist progress

## Output Requirements

Deliver:
- 3 visual directions max
- one recommended direction
- token ideas
- component language
- trust component patterns
- CTA hierarchy
- notes on what should become reusable system elements
